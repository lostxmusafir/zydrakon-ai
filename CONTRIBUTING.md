# Contributing to Zydrakon AI

Thank you for your interest in contributing to Zydrakon AI! We welcome community contributions to help improve our developer tools, documentation, and website.

## How to Contribute

### 1. Reporting Bugs
- Check the open issues to see if the bug has already been reported.
- If not, open a new issue describing the bug, including steps to reproduce, expected behavior, and your system configuration.

### 2. Suggesting Enhancements
- Open an issue outlining the proposed enhancement, why it is useful, and how it should work.

### 3. Pull Requests
1. Fork the repository and create your branch from `develop`.
2. Ensure your code follows our TypeScript and formatting guidelines.
3. Commit your changes using Conventional Commits.
4. Open a pull request against the `develop` branch.

## Conventional Commits

We follow the Conventional Commits specification:
- `feat:` for new features and components.
- `fix:` for bug fixes.
- `docs:` for documentation updates.
- `chore:` for settings, config updates, and package dependencies.
- `style:` for changes that do not affect code logic (formatting, spacing).
- `refactor:` for code restructuring without feature modification.
